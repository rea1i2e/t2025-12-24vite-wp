# 開発ガイド

このセクションは、**開発者向け**のドキュメントです。開発中に頻繁に参照する情報をまとめています。

## このセクションのドキュメント

- **[architecture.md](architecture.md)**: 設計判断・守るルール（MUST/SHOULD/MAY）
  - プロジェクトの設計判断と守るべきルール
  - Vite連携の詳細など
  - ファイル構成の一覧は [overview.md](../04-project-specific/overview.md) に記載

- **[coding-standards.md](coding-standards.md)**: コーディング規約
  - PHP、Sass、JavaScriptのコーディング規約
  - 開発時に参照する規約

- **[development.md](development.md)**: 開発フロー・スクリプト
  - 開発時の作業フロー
  - npmスクリプトの使い方
  - ディレクトリ構成

## 読む順番

1. **[architecture.md](architecture.md)**: まず設計判断とルールを理解する
2. **[coding-standards.md](coding-standards.md)**: コーディング規約を確認する
3. **[development.md](development.md)**: 開発フローを理解する
