# 9. Vite開発時のポート5173運用

**ステータス**: 承認済み

## コンテキスト

`ty_vite_is_dev()` は `TY_VITE_DEV_SERVER`（既定では `http(s)://localhost:5173`）の `/@vite/client` へ HTTP HEAD し、応答が取れれば **dev 扱い**とする。したがって **同一端末の別プロジェクトが 5173 で Vite を動かしているだけでも**、このテーマ側は dev 判定になりうる。

代替として、案件ごとに専用ポート（例: 5199）へ寄せ、`TY_VITE_DEV_SERVER` と Vite の `server.port` を揃える案もある。一方でテンプレート既定は 5173 のままにし、運用で衝突を解消する選択も可能である。

## 決定

- **開発時の Vite は既定の 5173 を前提**とする（`vite.config`・`TY_VITE_DEV_SERVER` のデフォルトと整合させる）。
- **5173 が他プロセスに占有されている、または別案件の Vite が 5173 で動いている場合は**、必要に応じて **そのプロセスを止める**（または該当ターミナルを終了する）運用とする。
- 専用ポートへ切り替える方針は **採用しない**（現時点）。将来ポートを分ける場合は `TY_VITE_DEV_SERVER` と Vite 設定の両方を必ず揃える別 ADR で扱う。

実作業の手順・症状別の切れば箇所は、重複を避けるため次に集約する。

- `docs/01-development/development.md`（日常運用）
- `docs/03-troubleshooting/troubleshooting.md`（トラブル時）

## 結果

- **メリット**: ポート番号の二重管理が不要になり、テンプレートクローン直後の迷いが少ない。
- **デメリット・注意**: 複数 Vite を同時に 5173 で載せられない。**必ず占有プロセスを特定してから終了**すること（誤って別アプリを kill しない）。複数案件を同時にフロント開発する場合は、作業切り替え時に 5173 を空ける必要がある。
