# Architecture Decision Records (ADR)

このディレクトリには、プロジェクトの重要な意思決定をADR形式で記録します。

## ADRとは

ADR（Architecture Decision Records）は、アーキテクチャに関する重要な意思決定を記録するための形式です。各意思決定について、以下の情報を記録します：

- **タイトル**: 意思決定の内容
- **ステータス**: 提案中/承認済み/非推奨/置き換え済み
- **コンテキスト**: 意思決定が必要になった背景
- **決定**: 採用した解決策
- **結果**: 決定による影響や結果

## ファイル命名規則

各ADRファイルは以下の形式で命名します：

```
NNNN-タイトル.md
```

- `NNNN`: 4桁の連番（例: `0001`, `0002`）
- `タイトル`: 意思決定の内容を表す短いタイトル（ハイフン区切り）

例:
- `0001-vite-integration.md`
- `0002-wordpress-theme.md`
- `0003-function-prefix.md`

## テンプレート

新しいADRを作成する際は、以下のテンプレートを使用してください：

```markdown
# N. タイトル

**ステータス**: 提案中/承認済み/非推奨/置き換え済み

## コンテキスト

意思決定が必要になった背景を記述します。

## 決定

採用した解決策を記述します。

## 結果

決定による影響や結果を記述します。

## 関連ADR

関連する他のADRがあれば、ここに記載します。
```

## 既存の意思決定

既存の意思決定については、`docs/06-reference/migration-log.md` を参照してください。必要に応じて、重要な意思決定をADR形式に変換してこのディレクトリに追加してください。
