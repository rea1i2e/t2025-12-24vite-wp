# 2. ナビゲーション項目のURL取得方法

**ステータス**: 承認済み

## コンテキスト

`func-nav-items.php` では、ナビゲーションメニューの項目を配列で定義し、`ty_get_nav_items()` で返す設計になっています。これは主にヘッダーやフッターなどのメニューでループ出力する前提です。

特定のページ（例: お問い合わせページ）のURLを、ボタンやテキストリンクの `href` 属性として直接取得したいケースが発生しました。

以下の2つのアプローチを検討しました：

1. **新しい関数を追加**: `ty_get_nav_item_url($slug)` のような関数を作成し、ナビゲーション定義から検索してURLを返す
2. **既存関数を直接使用**: `ty_get_page($slug)` を直接使用する

## 決定

**既存の `ty_get_page()` を直接使用する方針を採用しました。**

理由：

- `ty_get_nav_item_url($slug)` を追加しても、実質的に `ty_get_page($slug)` と同じ結果になる
- ナビゲーション定義に依存するだけで、追加のメリットがない
- `ty_get_page()` は既に存在し、ナビゲーション定義に含まれていないslugでも使用可能（柔軟性が高い）
- 関数呼び出しが1回で済む（パフォーマンス的にも微差だが有利）

## 結果

特定のページのURLが必要な場合は、以下のように `ty_get_page()` を直接使用します：

```php
// 通常のページ
<a href="<?php echo ty_get_page('contact'); ?>">お問い合わせ</a>

// トップページ
<a href="<?php echo ty_get_page(); ?>">トップ</a>
```

`func-nav-items.php` は現状のままで、配列を返す関数としての役割に専念します。特定のページのURLが必要な場合は、`ty_get_page()` を直接使用する方針としました。
