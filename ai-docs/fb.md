# 修正 FB（一括指示用）

案件中の修正指示をここに箇条書きで溜め、まとまったら AI に **1回** で渡す。  
「1件直して → 待つ → また1件」のラリーを減らすための作業メモ（Git にコミットしてよい雛形。案件ごとに中身を書き換える）。

## 使い方

1. 目視・ディレクター指摘・`figma-qa-compare` の「修正依頼文」を **未対応** に追記する
2. ある程度溜まったら、下のプロンプトをチャットへ貼る
3. 直したものはチェックを付けるか **対応済** へ移す
4. 汎用ルールとして残したい指摘だけ、別途 `ai-code-feedback-stock`（ナレッジ）へストックする

## 役割分担

| 置き場 | 用途 |
|--------|------|
| **本ファイル（`ai-docs/fb.md`）** | この案件の実装を直す一時リスト |
| **`qa-reports/*.md`** | 初稿前 QA の差分レポート（転記元） |
| **ナレッジ `ai-code-feedback-stock`** | 横断ルール化したい指摘の記録（修正指示そのものではない） |

## ルール

1. 未対応の項目だけ着手する（対応済は触らない）
2. 1ラリーでまとめて直す。項目ごとにチャットを分けない
3. HTML/EJS/PHP のテキストコンテンツは自動生成しない（既存 or ユーザー提供を正）
4. 直した項目は本ファイルでチェック済みにするか「対応済」へ移す
5. 判断が必要なもの・デザイン未確認のものは直さず「要確認」として残す
